* Format of storing schema info:

* The format of schema description file will be .zn0w_sch

* The file's structure:
Schema_name
Authorization (if none provided (default) then it will be given to the current user)
Character set encoding (utf8 by default)
File path of the schema data (the actual data in the form of it's tables)
* Meta-data:
TABLE
	NAME table_name
		COLUMN col_name col_datatype (col_conditions, col_default_values, ...)
		COLUMN ...
END
TABLE
	...
END
...
* End of file

* Character set encoding possible values: (will be appended in the development process)
	utf8 - UTF-8 Unicode

* Column conditions and default values (attributes): (will be appended in the development process)
	notnull  - the field cannot be null
